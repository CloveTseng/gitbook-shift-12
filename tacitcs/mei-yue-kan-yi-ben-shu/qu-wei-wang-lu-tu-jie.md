#### URL 與 URI 的區別

* URI：電話這個品項
* URL：你辦公桌上的電話

一個標準的 URL 包含五點：

1. 協定 ( `http://` 或 `https://` )
2. 域名
3. 連結埠 ( `:80` 或 `:443` )
4. 虛擬目錄 (路徑結構)
5. 文件名 (檔案名稱)

#### HTTP 的請求 (Request) 和響應 (Response)

* 常見請求

| 請求方法   | 語義定義  | 冪等性 | 對伺服器資源的戰略影響             |
| ------ | ----- | --- | ----------------------- |
| GET    | 獲取資源  | 是   | 資源唯讀操作，應被高度快取。          |
| POST   | 提交數據  | 否   | 非冪等。重複提交可能導致重複扣款等副作用。   |
| HEAD   | 獲取首部  | 是   | 用於探針檢測（檢查檔案大小或存續），節省流量。 |
| PUT    | 更新/創建 | 是   | 具備覆蓋語義，適合特定 ID 資源的更新。   |
| DELETE | 刪除資源  | 是   | 移除特定標識資源。               |

* 常見狀態碼
  * 1xx (指示)：請求已接收，常見於協議切換
  * 2xx (成功)：`200 OK` 代表標準成功；`206 Partial Content` 則是高效能影音網站實現分段下載與斷點續傳的關鍵
  * 3xx (重定向)：`307 Temporary Redirect` 常見於 HTTP 轉 HTTPS 的安全性遷移
  * 4xx (客戶端錯誤)：除了 `404`，應關注 `499 Client Closed Request`。這代表客戶端等不及響應便主動斷開，通常暗示後端處理過慢
  * 5xx (服務端錯誤)：系統異常的最高警報，尤其是 `502 Bad Gateway`

#### Socket 是什麼？

將數據從 A 發到 B 時，socket 就扮演了端點文件包的角色，兩台電腦如果要通話的話，就各自需要一個 socket 插頭，並且這個 socket 插頭封裝了收發資訊的方法 (TCP 或 UDP)
