# 新鳥前端職場生存計畫

`w1 ~ w12`

<details>

<summary>生存與適應</summary>

* 了解公司文化與組織架構
  * 搞清楚「誰是誰」： 不只是技術主管，還要知道 PM（產品經理）、QA（測試）以及核心開發者是誰。觀察遇到問題時，大家傾向找誰討論。
  * 同步溝通頻率： 了解團隊使用 Slack、Discord 或 Teams 的習慣。觀察大家在什麼時候會發起討論，非同步溝通的節奏如何。
  * 閱讀內部文件： 找尋公司的 Wiki、Notion 或 Confluence。了解公司的產品願景與核心價值，這能幫助你在寫代碼時更理解業務邏輯（Business Logic）。
* 理解公司技術架構與開發流程
  * 熟悉公司的環境建置習慣與 git flow
  * CI/CD 流程
  * TypeScript：專案怎麼定義 `type`&#x20;
  * Vue：觀察專案的目錄架構（`components/`, `composables/`, `middleware/`），注意團隊如何拆解 SFC 以及 Composition API 的封裝邏輯
  * API 串接： 檢查專案中 `axios` 或 `ohmyfetch` 的封裝方式，了解攔截器（Interceptors）如何處理 Token 或報錯。
* 獨立開發能力
  * 爭取拿到獨立開發專案
  * 善用 Obsidian： 將公司專案中常見的 `type` 定義方式、特殊的開發規範記錄在 Obsidian 中，建立自己的「公司技術知識庫」。



</details>
